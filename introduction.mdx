# The Fork

> One decision split your life in half. This app lets you talk to the version of you who took the other path.

## What It Is (and What It Isn't)

**What it is:**
- A stateless, single-session chat that lets you talk to "Other You" - an LLM persona that role-plays the life you didn't choose
- An exploration of alternate paths, consequences, and the weight of decisions
- A blunt mirror that reflects lived consequences and tradeoffs

**What it isn't:**
- Therapy or professional mental health support
- A confidant that stores your secrets (sessions are ephemeral)
- A soft place to land by default - expect honesty over comfort

**Tone:** Choose an intensity (mild / savage / brutal). The persona stays in-character and reflects lived consequences, tradeoffs, and blunt observations.

<Warning>
**Trigger Warning:** Conversations can surface uncomfortable material. If you're in crisis, stop now and seek help.

Crisis resources:
- **U.S./Canada:** 988 (call/text) - immediate help
- **U.K. & ROI:** Samaritans 116 123
- For other locations, check your local emergency resources
</Warning>

## How It Works

<Steps>
  <Step title="Define Your Fork">
    You provide a fork statement - a single defining decision and the path you didn't take.
    
    Example: *"I chose engineering instead of art"* or *"I moved to the city instead of staying in my hometown"*
  </Step>
  
  <Step title="Pick Your Intensity">
    Select the tone level:
    - **Mild:** Supportive, reflective, grounded. Still honest.
    - **Savage:** Blunt, truth-forward, calls out avoidance and self-deception.
    - **Brutal:** No comfort, no flinching. Extremely direct.
  </Step>
  
  <Step title="Start Chatting">
    The frontend posts your session (sessionId + messages + forkStatement + intensity) to the backend.
  </Step>
  
  <Step title="AI Response">
    The backend composes a system prompt (the "Other You" persona + safety checks + style mirroring), calls the Emergent LLM, and returns a reply.
  </Step>
  
  <Step title="Ephemeral by Design">
    Conversations are not stored. MongoDB exists in the template but is optional and unused for chat functionality.
  </Step>
</Steps>

## Safety & Ethics

The app intentionally pushes hard at lived choices. The server implements:

<CardGroup cols={2}>
  <Card title="Self-Harm Detection" icon="life-ring">
    Immediate refusal with crisis resources when self-harm markers are detected in messages.
  </Card>
  
  <Card title="Hate Speech Blocking" icon="ban">
    Refusal responses for dehumanizing content and hate markers.
  </Card>
  
  <Card title="Character Boundaries" icon="user-shield">
    Strict "stay in character" rules with explicit prohibition on slurs, threats, and identity attacks.
  </Card>
  
  <Card title="Intensity Guardrails" icon="sliders">
    Controls allow profanity but never abuse - even at brutal intensity.
  </Card>
</CardGroup>

<Note>
If you plan to use this beyond private experiments, get an ethics review, add moderation, and be explicit about limitations to users.
</Note>

## Architecture Overview

The Fork uses a simple client-server architecture:

```
┌─────────────┐         ┌─────────────┐         ┌──────────────┐
│             │  HTTP   │             │   API   │              │
│   React     │────────>│   FastAPI   │────────>│  Emergent    │
│   Frontend  │  /chat  │   Backend   │         │     LLM      │
│             │<────────│             │<────────│              │
└─────────────┘         └─────────────┘         └──────────────┘
                              │
                              │ (optional)
                              v
                        ┌─────────────┐
                        │   MongoDB   │
                        │ (template)  │
                        └─────────────┘
```

### Key Components

**Backend (`backend/server.py:301-369`)**
- `POST /api/chat` - Main chat endpoint
- Safety checking functions
- System prompt generation with intensity controls
- Style mirroring logic to match user's writing mechanics

**Frontend**
- `ForkSetup.js` - Initial fork statement input
- `ChatWindow.js` - Conversation interface
- `IntensityToggle.js` - Intensity selector
- `MessageBubble.js` - Message display component

## Project Philosophy

From the README:

> If you came here to ask "what if?", do it honestly. The Fork won't make it gentle for you; it will make it real. If you're hacking on it, keep things safe, include tests, and respect the persona rules baked into server.py.
>
> Open the door carefully.

This is a tool for introspection, not escapism. The design choices - from ephemeral sessions to intensity controls to built-in safety measures - reflect a balance between psychological challenge and responsible deployment.
